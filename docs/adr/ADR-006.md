# ADR-006: Data Provenance and Synthetic-Label Policy

## Status
Proposed

## Context
Fusing synthetic simulators with real urban data creates mixed-provenance graphs. If synthetic labels or records are treated identically to observed sensor values, model validation fails to reflect actual performance.

## Decision
- Every record type (`NodeRecord`, `EdgeRecord`, `FuzzyState`, etc.) must include an explicit boolean flag: `is_synthetic`.
- Synthetic records must contain metadata describing the generating simulator version and input seed.
- Production validation runs must segment metrics by `is_synthetic` to prevent synthetic bias in metrics reporting.

## Consequences
- Prevents mixing simulated targets with real targets during validation.
- Enforces clear traceability and transparency of data sources.
