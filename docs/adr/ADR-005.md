# ADR-005: Research and Production Profiles

## Status
Proposed

## Context
Deploying ML models requires balancing diagnostic depth with operational safety. Research tasks require exhaustive diagnostic tracing and configuration replays, whereas production calls demand bounded latency, authenticated APIs, and no synthetic fallbacks.

## Decision
We define two operating profiles:
1. `research`: Allows deterministic experiments, exact config replay, exhaustive diagnostics, and slow reference implementations.
2. `production`: Enforces validated artifacts, bounded latency, failure isolation, and strict failure on external-adapter issues (no silent synthetic fallback).

## Consequences
- Watchdog behavior changes based on active profile.
- Outages or database failures in production trigger alerts instead of silently returning mock data.
