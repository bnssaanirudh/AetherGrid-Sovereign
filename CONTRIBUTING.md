# Contributing

Thanks for helping improve AetherGrid-Sovereign.

## Style
- Follow PEP8 and keep lines under 100 characters.
- Use type hints for public functions and classes.
- Prefer clear, descriptive variable names over abbreviations.

## Testing
- Run `python -m pytest -v` before opening a PR.
- Mark slow tests with `@pytest.mark.slow`.
- Avoid network calls in unit tests.

## Commits
- Use concise commit messages.
- Keep unrelated changes in separate commits.
